디자이너 1주 → 30분 셀프

워크샵·이벤트마다 디자이너에게 의뢰하던 랜딩을, AI 카피 마법사와 미리 만든 섹션 디자인 조합으로 본인이 직접 30분에 만들고 톤·디자인 일관성까지 유지하게 된 이야기.

01

왜 만들었나

매월 워크샵 운영하는데 랜딩을 매번 외주 — 한계 명확

실제 운영 상황
"워크샵·이벤트를 자주 열다 보니 — 매번 디자이너에게 카피·디자인 의뢰 → 1주 대기 → 비용 → 톤 검수 → 수정 → 게시. 같은 브랜드인데 매 랜딩마다 톤·디자인이 미세하게 다르고, 비용도 누적. 월 N건 워크샵이면 외주만 N주 + 매월 X만원 고정 지출."

대표 페인 포인트

  • 디자이너에게 맡기면 1주 이상 기다림 — 워크샵 마지막 순간 변경 안 됨
  • 매번 디자인 비용 누적 — 워크샵 자주 열면 매월 N건 × 비용
  • 같은 브랜드인데 랜딩마다 톤이 미묘하게 다름
  • 코드 직접 만지기 부담 — 1인 사업자가 디자인까지 하면 본업 시간 부족
  • 디자이너 바뀌면 톤이 또 흔들림

보충 필요 처음 만들기 시작한 구체적 계기

02

어떻게 풀었나

실제 겪었던 시행착오

1
AI에게 카피 1개만 받음 — 마음에 들 때까지 계속 다시 호출
처음에는섹션마다 카피 1개만 만듦 → 마음에 안 들면 또 호출 → 또 다시. 톤·표현 비교가 안 되고 시간 낭비
→
바꿨더니한 번 호출에 카피 3안 동시 생성. 미리보기로 비교 → 1개 선택 또는 직접 편집. 결정 시간이 크게 줄어듦
2
섹션마다 색·디자인을 따로 결정 — 톤이 흔들림
처음에는히어로·문제 제기·차별점 등 섹션마다 색·여백을 그때그때 결정. 같은 워크샵 랜딩인데 위아래 색이 안 맞음
→
바꿨더니첫 화면에서 테마 색만 한 번 결정 → 모든 섹션에 자동 적용. 디자이너 없이도 일관성 유지
3
랜딩마다 톤이 흔들림 — 학습이 안 됨
처음에는각 랜딩이 독립이라 이전에 좋았던 표현·구조가 사라짐. 매번 처음부터 만듦
→
바꿨더니확정한 카피를 모아두고 다음 카피 만들 때 자동 참고 자료로 AI에게 전달. "이런 톤으로 써줘"가 자동으로 들어감. 사용할수록 브랜드 톤 일관성 ↑
4
코드 직접 만지기는 너무 부담
처음에는워크샵마다 HTML 코드를 손으로 작성 — 코드 모르는 운영자에게 큰 진입 장벽
→
바꿨더니워크샵 랜딩 필수 8개 섹션(첫 인상·문제 제기·해결책·차별점·커리큘럼·준비물·신청·자주 묻는 질문)을 미리 디자인해두고 텍스트만 채움
5
일반 AI 카피는 워크샵 톤에 약함
처음에는보통 AI가 만드는 카피는 워크샵·교육 톤에 약함. 형식적·일반론이 많아 매번 수정
→
바꿨더니워크샵 카피 규칙(브랜드 원칙·금지 표현·섹션별 작성법)을 정리해 AI에게 매번 자동 전달. 워크샵 페르소나에 맞는 톤으로 생성

보충 필요 다른 시행착오가 있다면

03

실제 동작 화면

섹션마다 카피 3안 → 선택 → 미리보기 → 확정

섹션별 AI 카피 3안 → 미리보기 → 확정

섹션 1개당 약 3분 — 8개 섹션 = 30분 안에 랜딩 완성

섹션마다 카피 3안 동시 생성 → 1개 선택 또는 직접 편집 → 미리보기로 즉시 확인 → 확정. 확정된 카피는 자동 저장되어 다음 랜딩의 톤 참고 자료가 됨. 마지막 단계에서 섹션별 HTML 9개 출력 → 아임웹·노션·자사몰 어디든 그대로 복붙.
