5개 도구 → 1개 흐름

워크샵 1번 알림톡 보내려고 노션·ChatGPT·솔라피·스티비·탈리를 오가던 운영자가, 한 화면에서 글 만들고·검수 받고·발송하고·노션 결과까지 정리하게 된 이야기.

01

왜 만들었나

1인 브랜드 운영자의 5개 도구 분산 문제

실제 운영 상황
"워크샵 신청자에게 알림톡 1번 보내려면 — 노션에서 신청자 정리 → ChatGPT로 글 쓰기 → 카카오 규칙 다시 검색 → 솔라피 화면에 변수 넣고 등록 → 검수 통과 확인 → 솔라피에서 직접 발송 → 다시 노션 가서 누구한테 갔는지 적기. 매번 30분 이상이 들었어요."

대표 페인 포인트

  • 보낼 때마다 카카오 검수 규칙을 다시 검색 — 어떤 단어가 광고로 걸리는지·글자 수는 몇 자까지인지 매번 확인
  • 알림톡과 친구톡이 만드는 방식이 완전 다름 — 친구톡만 8가지 형식인데 각각 규칙이 또 다름
  • 같은 항목인데 노션·솔라피·입력 폼 이름이 따로 — 한 번 오타 나면 발송 자체가 안 됨
  • 발송이 끝나도 "누구한테 갔는지"는 노션에 손으로 다시 적어줘야
  • 실패한 사람을 보려면 솔라피 화면을 또 따로 켜야 — 한 흐름이 안 됨

보충 필요 처음 만들기 시작한 구체적 계기·시기

02

어떻게 풀었나

실제 겪었던 시행착오

1
하나의 AI에 모든 규칙을 다 넣었더니 결과가 흐릿함
처음에는알림톡·친구톡 8가지 형식 규칙을 한 AI에 다 넣었더니, 워크샵 안내인데 마케팅 톤이 나오고 마케팅인데 워크샵 톤이 나옴. AI가 헷갈려서 결과가 흐릿
→
바꿨더니용도별로 AI를 5개로 분리 — 알림톡 전용, 친구톡 자유형 전용, 친구톡 템플릿형 전용, 알림톡 검수 전용, 친구톡 검수 전용. 각 AI가 자기 영역만 보니까 결과가 또렷해짐
2
친구톡 8가지 형식 — 규칙이 솔라피 문서 곳곳에 흩어져 있음
처음에는친구톡도 알림톡처럼 한 가지인 줄 알았더니 8가지 형식이 각각 다름. 특히 "캐러셀 상품" 형식은 "위쪽에 버튼 넣으면 안 되고 항목 안에만 가능" 같은 규칙이 문서 여기저기 흩어져
→
바꿨더니8가지 각각의 입력 양식을 미리 코드로 만들어두고 자동 변환. 사용자는 형식만 고르면 끝
3
솔라피가 카카오 검수 신청까지는 자동으로 안 해줌
처음에는솔라피가 알림톡 등록은 자동인데 "이거 검수해 주세요"는 사용자가 솔라피 화면에 직접 들어가서 클릭해야. 자동화 흐름이 끊김
→
바꿨더니솔라피가 공식 안내한 통로 말고 내부 주소를 찾아내서 자동 신청. 등록 → 검수 → 통과까지 알림톡 안에서 한 번에
4
같은 항목인데 이름이 3개 — 매번 헷갈림
처음에는노션은 "이름", 알림톡은 "user_name", 입력 폼은 "성함" — 한 항목인데 이름이 3개. 매번 매핑해줘야 하고 오타 한 번이면 안 보내짐
→
바꿨더니이름 하나로 통일 — 폼에 "이름"이라 쓰면 노션도 "이름", 변수도 "이름". 발송 유형(워크샵·콘텐츠·마케팅)별 기본 항목도 자동으로 채워줌
5
검수 결과가 너무 늦게 떠서 답답
처음에는광고 단어·글자 수 검사와 AI 의미 평가를 한 번에 처리. 8~12초 빈 화면 대기. 사용자가 끊긴 느낌
→
바꿨더니빠른 검사는 즉시 표시, AI 평가는 글이 흘러나오듯 점진 표시. 사용자는 결과 일부를 바로 보고 나머지를 기다림
