(1) AI 카피 생성·검수

Claude Code가 매번 카피 룰 외울 일을 없앴습니다

사용자는 어떤 발송 유형인지(워크샵·콘텐츠·마케팅·직접)만 고르면 됩니다. 5개 AI 에이전트가 알림톡·친구톡 8형식을 자동으로 구분하고 카피를 만듭니다. 한 번 저장해둔 브랜드 톤·보이스·금지 단어·CTA 스타일이 모든 카피 생성에 자동으로 주입되어 매번 같은 브랜드 톤이 유지됩니다. 광고 표현·글자수·변수 형식·머리말 규칙은 즉시 검사하고, AI 톤·의미 평가는 글이 흘러나오듯 점진적으로 표시됩니다.

[스크린샷 자리 — Claude Code 카피 생성·검수 화면 (브랜드 설정 탭 포함)]

핵심 동작

  • 5개 AI 에이전트 자동 선택 (알림톡·친구톡 자유형/템플릿형 + 검수 2종)
  • 브랜드 프로필 자동 주입 — 톤·보이스·페르소나·금지 단어·CTA 스타일
  • 광고·글자수·변수·머리말 즉시 검사 (수십 가지 카카오 규칙 자동)
  • AI 톤·의미 평가 스트리밍 (사용자 대기 시간 단축)
  • 발송 유형별 자유 필드 자동 세팅 (이름 = 변수 = 노션 컬럼명)
(2) 등록·발송

솔라피 콘솔에 들어갈 일이 없어요

검수 통과된 카피는 솔라피에 자동 등록되고 카카오에 검수 신청까지 자동으로 요청합니다. 발송 캠페인 모달에서 즉시·예약·자동 중 선택만 하면 됩니다. 노션에서 받는 사람을 필터로 고르고, 시점별 변수 매핑까지 한 모달 안에서 끝납니다.

[스크린샷 자리 — 솔라피 캠페인 5단계 모달]

핵심 동작

  • 솔라피 자동 등록 + 카카오 검수 자동 신청
  • 발송 캠페인 5단계 마법사 (템플릿·노션·필터·발송·콜백)
  • 즉시 발송 / 예약 / 자동(정기) 중 선택
  • 실패한 사람까지 결과 화면 한눈에
(3) 결과 정리

노션에 손으로 결과를 적을 일이 없어요

발송이 끝나면 노션의 row가 자동으로 업데이트됩니다. 발송 시각·결과·재시도 여부까지 사용자가 매핑한 컬럼에 따라 자동 정리. 풀 스냅샷이 영구 아카이브로 저장되어 다음 카피의 학습 데이터로도 활용됩니다.

[스크린샷 자리 — 노션 결과 자동 정리]

핵심 동작

  • 발송 성공·실패 row 자동 업데이트 (사용자 매핑 컬럼)
  • 영구 아카이브 저장 (output/complete/ 풀 스냅샷)
  • 다음 카피의 학습 데이터 1차 소스 (반려·승인 패턴)
Chapter 2.5

만들면서 배운 것 — 실제 시행착오

한 번에 잘 된 게 아닙니다. 5가지 큰 시행착오를 거쳐 지금 구조가 나왔어요.

1
하나의 AI에 모든 규칙을 다 넣었더니 결과가 흐릿함
처음에는알림톡·친구톡 8가지 형식 규칙을 한 AI에 다 넣었더니, 워크샵 안내인데 마케팅 톤이 나오고 마케팅인데 워크샵 톤이 나옴. AI가 헷갈려서 결과가 흐릿
→
바꿨더니용도별로 AI를 5개로 분리 — 알림톡 전용, 친구톡 자유형 전용, 친구톡 템플릿형 전용, 알림톡 검수 전용, 친구톡 검수 전용. 각 AI가 자기 영역만 보니까 결과가 또렷해짐
2
친구톡 8가지 형식 — 규칙이 문서 곳곳에 흩어져 있음
처음에는친구톡도 알림톡처럼 한 가지인 줄. 특히 "캐러셀 상품" 형식은 "위쪽에 버튼 넣으면 안 되고 항목 안에만 가능" 같은 규칙이 문서 여기저기 흩어져
→
바꿨더니8가지 각각의 입력 양식을 미리 코드로 만들어두고 자동 변환. 사용자는 형식만 고르면 끝
3
솔라피가 카카오 검수 신청까지는 자동으로 안 해줌
처음에는솔라피가 알림톡 등록은 자동인데 "이거 검수해 주세요"는 사용자가 솔라피 화면에 직접 들어가 클릭해야. 자동화 흐름이 끊김
→
바꿨더니솔라피가 공식 안내한 통로 말고 내부 주소를 찾아내서 자동 신청. 등록 → 검수 → 통과까지 한 번에
4
같은 항목인데 이름이 3개 — 매번 헷갈림
처음에는노션은 "이름", 알림톡은 "user_name", 입력 폼은 "성함" — 한 항목인데 이름이 3개. 오타 한 번이면 안 보내짐
→
바꿨더니이름 하나로 통일 — 폼에 "이름"이라 쓰면 노션도 "이름", 변수도 "이름". 발송 유형별 기본 항목도 자동 채워줌
5
검수 결과가 너무 늦게 떠서 답답
처음에는광고 단어·글자 수 검사와 AI 의미 평가를 한 번에 처리. 8~12초 빈 화면 대기. 사용자가 끊긴 느낌
→
바꿨더니빠른 검사는 즉시 표시, AI 평가는 글이 흘러나오듯 점진 표시. 사용자는 결과 일부를 바로 보고 나머지를 기다림

(2) 운영자 인터뷰

Q. 도입 전 가장 큰 어려움은 무엇이었나요?
보충 필요 예: "매번 카카오 검수 규칙을 다시 검색해야 했어요. 광고 단어로 걸리면 또 처음부터 카피를 수정해야 했고, 솔라피 콘솔에 들어가 변수 매핑하는 것도 매번 번거로웠어요."
Q. 도입 후 가장 크게 달라진 점은?
보충 필요 예: "한 화면에서 카피·검수·발송이 다 끝나요. 솔라피 콘솔이나 노션을 다시 안 들여다봐도 결과가 자동으로 정리돼서 본업에 더 집중할 수 있어요."
Q. 브랜드 톤 일관성은 어떻게 유지되나요?
보충 필요 예: "한 번 브랜드 톤·금지 단어·CTA 스타일을 저장해두면 매번 카피 생성에 자동으로 들어가요. 워크샵 안내인지 마케팅인지에 따라 톤이 자동으로 분리되니까 글마다 톤이 흔들리지 않아요."
Q. 어떤 사업자에게 추천하고 싶나요?
보충 필요 예: "워크샵·이벤트를 자주 여시는 1인 운영자분들. 한 달에 알림톡 5건만 보내도 누적되는 시간 차이가 정말 크거든요."
