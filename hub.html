매번 외우던 규칙 → 시스템에 맡기기

검색 노출 잘 되는 글 쓰는 법·아임웹 본문 만드는 법·브랜드 톤을 매번 머리로 외우던 운영자가, 한 번 정리해두고 글마다 자동으로 적용되게 만든 이야기.

01

왜 만들었나

1인 사업자가 직접 SEO 블로그까지 운영하면서 생긴 문제

실제 운영 상황
"자사몰 블로그(실행 연구소)에 SEO 글 1편을 쓰려면 — 키워드 조사 → 제목·소제목 구조 짜기 → 본문 → 아임웹에 들어갈 코드(HTML)로 변환 → 색·여백 일일이 확인 → 검수. 1편당 6시간이 매번. 외주는 톤이 안 맞고 비용도 부담."

대표 페인 포인트

  • 매번 검색 노출 잘 되는 글 쓰는 법을 다시 찾아봄 — 제목에 키워드 어디 넣어야 하는지, 설명문은 몇 자인지
  • 메인 글과 가지 글을 어떻게 다르게 써야 하는지 매번 헷갈림 — 분량·소제목 수·서로 어떻게 링크 거는지
  • 아임웹 본문은 HTML 코드 그대로 넣어야 — 챕터 번호 색·본문 줄간격 등 14항목 룰을 매번 검사
  • "알아보겠습니다", "이 글에서는~" 같은 SEO에 약한 표현을 자기도 모르게 씀
  • 글마다 "~합니다"와 "~요"가 섞여 톤이 흔들림

보충 필요 처음 만들기 시작한 계기·시기

02

어떻게 풀었나

실제 겪었던 시행착오

1
한 글에 너무 많은 키워드 — 오히려 검색 노출 약함
처음에는관련 키워드를 한 글에 다 넣으려다 분량 길어지고 키워드 밀도도 떨어짐. 글끼리 연결도 평면적
→
바꿨더니메인 글과 가지 글로 분리 — 메인은 큰 키워드(긴 글), 가지는 세부 키워드(짧은 글). 메인은 가지들에 링크 걸고, 가지는 메인으로 다시 링크. 검색엔진이 좋아하는 구조
2
SEO 규칙을 매번 머리로 외움 — 일관성이 안 맞음
처음에는제목에 키워드 어디 넣어야 하는지·서두 200자 안에 키워드 있어야 하는지·설명문은 120~160자인지 — 매번 검색해서 적용
→
바꿨더니이런 규칙을 AI에게 한 번 정리해서 알려줌. 다음부터 글 만들 때마다 자동 적용. 매번 외울 필요 없음
3
아임웹 HTML — 색·여백을 손으로 매번 다름
처음에는아임웹 본문은 코드 그대로 받음. 글마다 챕터 번호 색·본문 줄간격·팁 박스 색이 미묘하게 달라짐
→
바꿨더니HTML 17개 블록을 미리 만들어두고 텍스트만 갈아끼움. AI가 색·여백 14항목을 자동 검사
4
톤이 매번 흔들림 — 정보형·권유형 섞임
처음에는"~합니다"와 "~요"를 무작위 섞고 느낌표 남발. "알아보겠습니다" 같은 SEO에 약한 표현도 자주 나옴
→
바꿨더니정보 전달은 "~합니다", 응원·공감은 "~요"로 분리. 금지 표현 목록 정리. AI가 위반하면 빨강으로 표시
5
메타 정보를 주석으로 적었다가 본문에 노출되는 사고
처음에는설명문·주소를 HTML 주석(`<!-- -->`)으로 적었더니 아임웹에 올린 후 주석이 본문에 텍스트로 그대로 노출됨
→
바꿨더니주석 절대 금지로 규칙 추가. 메타 정보는 HTML 파일 맨 위에 일반 텍스트로 표시
