(1) 정보·테마 — STEP 0

기본 정보 1번 입력하면 모든 섹션이 자동 동기화

주제·타깃·일시·정원·참가비·진행 방식·마감일·탈리폼 ID·테마 색상을 한 화면에서 입력. 이후 모든 섹션 카피·HTML이 이 정보로 자동 생성·치환됩니다. 테마 색은 한 번만 결정하면 9개 섹션 전체에 일괄 적용되어 디자이너 없이도 일관성 유지.

[스크린샷 자리 — STEP 0 입력 화면]

핵심 동작

  • 워크샵 정보 9가지 한 화면 입력
  • 테마 색 1번 결정 → 모든 섹션 자동 치환
  • 탈리폼 ID 연결로 신청 자동 연동
  • 마감일 카운트다운 자동 생성
(2) 섹션별 카피 — STEP 1~8

Claude Code가 섹션마다 카피 3안 동시 생성

Hero·Painpoint·Bridge·Difference·Curriculum·Prepare·Apply·FAQ — 워크샵 랜딩 8 섹션마다 카피 3안이 한 번에 생성됩니다. 미리보기로 비교하고 1개 선택 또는 직접 편집. 한 번 저장해둔 브랜드 톤·금지 단어·이전 확정 카피가 자동으로 참고되어 사용할수록 톤 일관성이 올라갑니다.

[스크린샷 자리 — 섹션별 카피 3안 화면]

핵심 동작

  • 섹션 8개 각각 카피 3안 동시 생성
  • 미리보기 + 직접 편집 가능
  • 워크샵 카피 규칙 자동 주입 (브랜드 원칙·금지어)
  • 이전 확정 카피 5개 few-shot 자동 참고
(3) HTML 저장·발행 — STEP 9

섹션 HTML 9개 자동 저장 → 아임웹 복붙

STEP 9에서 0_countdown.html ~ 8_faq.html까지 섹션 HTML 파일이 자동 저장됩니다. 각 HTML은 이미 디자인된 베이스에 카피·테마 색상이 치환된 상태. 아임웹·노션·자사몰 어디든 인라인 HTML로 그대로 붙여넣기만 하면 발행 끝.

[스크린샷 자리 — HTML 9개 결과·아임웹 발행]

핵심 동작

  • 섹션별 HTML 9개 자동 저장
  • 테마 색·변수 모두 치환 완료 상태
  • 외주 의뢰 불필요 — 코드 손댈 일 없음
  • 확정 카피는 자동 archive (다음 카피 학습 데이터)
Chapter 2.5

만들면서 배운 것 — 실제 시행착오

디자이너 외주를 셀프로 바꾸면서 5가지 시행착오를 거쳤습니다.

1
AI에게 카피 1개만 받음 — 마음에 들 때까지 계속 다시 호출
처음에는섹션마다 카피 1개만 만듦 → 마음에 안 들면 또 호출 → 또 다시. 비교가 안 됨
→
바꿨더니한 번 호출에 카피 3안 동시 생성. 미리보기 비교 + 1개 선택. 결정 시간 크게 단축
2
섹션마다 색·디자인을 따로 결정 — 톤 흔들림
처음에는Hero·Painpoint·Curriculum 섹션마다 색·여백을 그때그때. 같은 워크샵인데 위아래 색이 안 맞음
→
바꿨더니STEP 0 테마 색 1번 결정 → 모든 섹션에 자동 치환. 디자이너 없이도 일관성 유지
3
랜딩마다 톤이 흔들림 — 학습이 안 됨
처음에는각 랜딩이 독립이라 이전에 좋았던 표현·구조가 사라짐. 매번 처음부터
→
바꿨더니확정 카피 저장·다음 카피에 자동 참고. 사용할수록 톤 일관성 ↑
4
코드 직접 만지기는 너무 부담
처음에는워크샵마다 HTML 코드 손으로 작성 — 코드 모르는 운영자에게 큰 진입 장벽
→
바꿨더니워크샵 필수 8 섹션 미리 디자인해두고 텍스트만 채움 (Hero·문제·해결·차별점·커리큘럼·준비물·신청·FAQ)
5
일반 AI 카피는 워크샵 톤에 약함
처음에는보통 AI 카피는 워크샵·교육 톤에 약함. 형식적·일반론 많아 매번 수정
→
바꿨더니워크샵 카피 규칙(브랜드 원칙·금지어·섹션별 작성법)을 정리해 AI에게 자동 전달

Q. 브랜드 톤은 어떻게 유지되나요?
보충 필요 예: "이전에 만든 랜딩 카피를 AI가 자동으로 참고해요. 사용할수록 우리 톤에 가까워지는 느낌이에요."
