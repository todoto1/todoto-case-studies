(1) 자동 수집 — GitHub Actions

PC가 꺼져 있어도 매시간 인스타 데이터 자동 수집

로컬 PC에서 매시간 스크립트를 돌리려면 PC가 항상 켜져 있어야 합니다. GitHub의 자동 시계(Actions cron)가 대신 매시간 인스타 Graph API를 호출하고 노션에 저장합니다. 외출·여행 중에도 데이터는 끊김 없이 쌓이고, 자정 누락 시 자동 복구 로직까지.

[스크린샷 자리 — GitHub Actions 자동 수집 로그]

핵심 동작

  • 매시간 자동 실행 (Meta Graph API 호출)
  • 전체 게시물 페이지네이션 (30일 이상은 영상·이미지 주소만 갱신)
  • 자정 누락 자동 복구 (22h 이상 경과 시 catch-up)
  • PC 켜져 있을 필요 없음
(2) 영구 저장 — Notion 4-DB

본인 노션에 누적 — 시계열 영구 보존, 락인 없음

Posts(게시물별 16컬럼) · AccountDaily(계정 단위 13메트릭) · Reports(인사이트 누적) · Demographics(지역·연령·성별 스냅샷) 4개 DB에 시계열로 누적. 각 브랜드가 본인 노션 워크스페이스에 저장해서 데이터 락인 없이 운영합니다. 인스타 자체 데이터가 1주 후 사라져도 우리 노션은 영구.

[스크린샷 자리 — 노션 4-DB 화면]

핵심 동작

  • 4 DB로 분리 저장 (게시물·계정·인사이트·데모)
  • 매시간 시계열 스냅샷 추가 (게시물별 변화 추적)
  • 본인 노션 워크스페이스 — 락인 없음
  • 캡션·영상 자막·이미지 텍스트까지 자동 추출 저장
(3) AI 분석 — Claude Code

AI 결론에 우리 메트릭 수치 인용 강제

일반 AI는 "스크롤 멈춤 유발", "이목 집중" 같은 콘텐츠 마케팅 격언만 반환합니다. Claude Code에는 "추측 금지·우리 수치 인용 강제" 규칙이 자동 주입되어 모든 결론에 "저장률 4.5%·평균 시청 22초·저장 35회" 식으로 우리 데이터를 인용합니다. 상위 25% 게시물의 후크·길이·이모지·CTA 패턴도 매주 자동 학습.

[스크린샷 자리 — Claude Code 인사이트 화면]

핵심 동작

  • 추측·일반론 금지 — 우리 메트릭 수치 인용 강제
  • 상위 25% 게시물 패턴 자동 추출 (매주 월요일)
  • 14개 분석 모듈 동적 조립 (주간·월간·연간·맞춤)
  • 분석 결과를 다음 카피 생성에 자동 참고 (자기강화)
Chapter 2.5

만들면서 배운 것 — 실제 시행착오

데이터 휘발·PC 의존·AI 일반론·노션 API 변화 — 5가지 시행착오를 거쳤습니다.

1
AI 분석이 일반론 — 우리 데이터를 안 봄
처음에는AI한테 "이 게시물 분석해줘" 했더니 "스크롤 멈춤 유발", "이목 집중" 같은 마케팅 격언만. 우리 좋아요·조회수·저장 수치 무시
→
바꿨더니AI에게 "추측 금지·우리 수치를 반드시 인용해라" 규칙 강제. 결과: "저장률 4.5%·평균 시청 22초" 식. 일반론 0건
2
PC가 꺼져 있으면 수집이 끊김
처음에는로컬 PC에서 스케줄러로 매시간 수집 → PC 잠들거나 끄면 데이터 누락. 외출·여행 후 며칠치 사라짐
→
바꿨더니GitHub의 자동 시계가 매시간 대신 수집. PC 꺼져 있어도 데이터는 노션에 쌓임
3
자정 데이터 수집이 누락됨
처음에는5/12 0시 데이터가 비어 있음. PC 슬립 + GitHub 시계 분 단위 부정확 + 우리 코드가 정확히 0시만 수집하도록 만들어짐
→
바꿨더니"마지막 수집 22시간 지났으면 시간대 상관없이 자동 수집"으로 보정. 누락 자동 복구
4
최근 50개만 갱신 — 옛 게시물 자료 사라짐
처음에는매일 최근 50개만 갱신. 30일 이상 된 게시물은 영상·이미지 주소가 만료되어 자막 추출 실패. 2023년 게시물부터 사라짐
→
바꿨더니모든 게시물 매일 갱신(157개). 30일 이상은 통계 skip + 영상·이미지 주소만 다시 받음 — 만료 회피
5
노션 카드 결과 영역이 짧고 표도 깨짐
처음에는AI 기획서를 노션 카드의 짧은 메모 자리에 저장. 2000자 제한·표 렌더링 안 됨. 노션 본문은 빈 상태
→
바꿨더니노션 카드 본문에 직접 저장. 제목·목록·표·구분선까지 정상 렌더링. "📝 todoto 기획서 — 시각" 헤더로 누적
Chapter 3

결과 — 휘발 → 영구, 감 → 메트릭 수치 기반

매주 데이터가 사라지던 운영이 영구 누적으로, AI 일반론이 메트릭 인용으로 바뀌었습니다.

(1) 운영 변화

기존에는 인스타 인사이트만 매주 보면서 감으로 판단했습니다. 이제는 자동 수집·영구 저장·메트릭 인용 분석이 한 흐름.

사용할수록 자기강화 데이터가 쌓이고, 콘텐츠 기획에도 잘 됐던 게시물의 후크·길이·이모지·CTA 패턴이 자동 반영됩니다.

(2) 운영자 인터뷰

Q. 도입 전 가장 큰 어려움은?
보충 필요 예: "게시물 1개가 시간에 따라 어떻게 떴는지가 안 보였어요. 100개 넘는 게시물인데 어떤 후크가 잘 됐는지 분석을 포기했어요."
Q. 도입 후 가장 크게 달라진 점은?
보충 필요 예: "감이 아니라 우리 수치로 판단하게 됐어요. AI한테 분석 시키면 '저장률 4.5%' 같은 우리 데이터를 인용해서 신뢰가 가요."
Q. 자기강화는 어떻게 작동하나요?
보충 필요 예: "매주 자동으로 잘 됐던 상위 25% 게시물의 패턴을 추출해줘요. 다음 콘텐츠 기획할 때 그 패턴이 자동으로 들어가서, 사용할수록 카피 품질이 올라가는 느낌."
Q. 어떤 사업자에게 추천하고 싶나요?
보충 필요 예: "인스타 운영 1년 넘으셨는데 데이터 분석 못 하시는 분들. 본인 노션에 다 쌓이니까 락인도 없어요."
