Instagram 운영을
노션 + Claude 통합 OS로

데이터는 GitHub Actions로 PC 꺼져도 매시간 수집. 인사이트는 로컬 Claude CLI가 메트릭 수치 기반으로만 해석. 사용할수록 카피 품질이 올라가는 자기강화 SaaS.

  • 하이브리드 — GH Actions 수집 + 로컬 Claude CLI 인사이트
  • "추측·일반론 금지" — 모든 결론에 메트릭 수치 인용 강제
  • 자기강화 — 상위 25% 게시물 패턴 자동 추출 → 다음 카피 가이드
  • 노션 4-DB가 1차 저장소 — Posts·AccountDaily·Reports·Demographics

인스타·노션·구글시트 → 1 대시보드

운영자가 인스타 인사이트·노션·구글시트를 따로 보며 추측에 의존하던 의사결정을, 매시간 자동 수집된 데이터 위에서 메트릭 수치 기반 인사이트로 바꾼 이야기. 6개월 운영 + Phase 17 진행 중.

01

왜 만들었나

1인 브랜드의 Instagram 운영 — 데이터 분산·휘발성·추측 기반 의사결정

실제 운영 상황
"인스타 게시물별 인사이트는 일주일 후 사라지고, 데모그래픽 변화는 추적 안 됨. 노션·구글시트 따로 정리하면 매번 손이 가고. 그래서 결국 "이 콘텐츠가 잘 됐다"는 감으로만 판단하게 돼요. 어떤 패턴이 통하는지 데이터로 안 보임."

대표 페인 포인트

  • 인스타 인사이트 휘발성 — 1주일 후 사라짐, 게시물별 시계열 추적 불가
  • 데모그래픽 변화 추적 X — 팔로워 city/country/age/gender 변화가 시간 흐름으로 보이지 않음
  • 어떤 콘텐츠가 통하는지 패턴 X — 100+ 게시물 누적인데 카피·시각·길이 패턴 추출 안 됨
  • AI 분석이 추측·일반론 — "스크롤 멈춤 유발", "이목 집중" 같은 콘텐츠 마케팅 격언만 → 신뢰 X
  • PC 꺼져있으면 수집 끊김 — 로컬 daemon만으로는 매시간 수집 보장 X

보충 필요 처음 만들기 시작한 구체적 계기 (예: "2025년 11월 todoto 인스타 ER 정체 + Meta Graph API 무료라는 발견 → 본격 빌드")

02

어떻게 풀었나

하이브리드 인프라 + 메트릭 강제 + 자기강화 시스템

GH Actions cron → 노션 4-DB → Claude CLI 인사이트 → 자기강화 패턴 → 콘텐츠 기획
1하이브리드 — PC 꺼져도 데이터는 쌓임
GitHub Actions cron이 매시간 Meta Graph 호출 + 노션 쓰기. 로컬 PC는 Claude CLI 인사이트·대시보드만 담당. 두 영역 완전 분리.
2노션 4-DB = 1차 저장소
Posts(16컬럼·게시물별) / AccountDaily(19컬럼·계정 단위 13메트릭) / Reports(인사이트 누적) / Demographics(4분류 스냅샷). 각 브랜드 본인 노션 워크스페이스에 저장 — 락인 X.
3AI 추측 금지 — 메트릭 수치 인용 강제
AGENT.md "절대 규칙" — "스크롤 멈춤 유발", "이목 집중", "효과적이다" 금지. 모든 인사이트에 "skip_rate 0.45·avg_watch 22초·saved 35회" 같은 우리 메트릭 인용 강제. 검증 완료.
4SKILL 14개 모듈 — 분석 시스템 분해
analysis-core(anti-speculation·baseline-comparison·metric-glossary) + insight-report 5종 + period-mode 5종(weekly/monthly/yearly/custom/all) + chat-rules. dispatcher가 입력 조건별 동적 조립.
5자기강화 시스템 — 매주 자동 학습
상위 25% 게시물의 후크·길이·이모지·CTA·발행 시점 패턴 자동 추출 → success_profile.json. 매주 월요일 새벽 자동 재계산. PC OFF 시에도 cloud worker로 처리.
6브랜드 컨텍스트 매핑 — 노션 자유 연결
사용자가 자기 노션 「브랜드 전략·디자인·제품 라인업」 DB를 카테고리에 매핑. Notion API 2025-09-03 부분 마이그레이션(multi-source DB 지원). 그룹·슬롯 자유 편집 + 섹션 anchor(heading/toggle 텍스트로 부분 추출) + 컨테이너 재귀 fetch.
7자정 수집 누락 catch-up — 3중 함정 해결
PC 슬립 + GH cron 분 단위 부정확 + 단순 매칭 3중 함정 → cloud_collect에 catch-up 로직(노션 Posts DB 최신 updated_at 기준 22h+ 경과면 자동 복구).
