매주 발표 슬라이드를
노션에서 한 번 골라 14장 자동

노션에 모아둔 자료가 슬라이드 14장에 자동 배치. 한 세션 6시간이 15분으로.

  • 노션 자료가 슬라이드 14장에 알아서 배치
  • 발표일·청중·목표가 한 번에 자동으로 채워짐
  • 발표 끝나면 슬라이드 주소가 노션에 자동으로 다시 적힘
  • 모바일로 미리 세션 정해두고 컴퓨터에서 슬라이드만 뽑으면 끝

매주 발표·웨비나·라이브를 하는 1인 사업자가, 슬라이드 14장을 매번 손으로 만들고·자료 복붙하고·발표 후엔 결과물이 흩어지던 흐름을, 노션에 자료만 모아두면 슬라이드·일정·다음 추천이 다 이어지는 시스템으로 바꾼 이야기.

핵심 동작

  • 노션 자료함·일정함이 슬라이드 입력
  • 세션 1개 클릭 → 제목·청중·자료·다음 추천 자동 채움

(2) 14장 생성 — Claude Code

14장 흐름 자동 구성 — 15분이든 60분이든 알아서

슬라이드 14장의 흐름을 발표 유형에 맞춰 자동 구성합니다: 인사 → 후킹 → 고민 짚기 → 공감 → 질문 → 컨셉 → 증거 → 숫자 → 사례 → 비포·애프터 → 시연 → 보상 예고 → 전환 → 다음 추천. 발표 길이에 따라 슬라이드별 시간이 자동으로 분배됩니다.

[스크린샷 자리 — 14장 자동 구성 화면]

핵심 동작

  • 14장 표준 흐름 자동 구성 (웨비나·워크샵·강의·라이브)
  • 길이별 자동 시간 분배 (15분 / 30분 / 60분)
  • 마지막 슬라이드에 다음 추천 세션 자동 삽입
  • 브랜드 톤·말투 자동 주입
(3) 자동 정리 — Notion

발표 끝나면 슬라이드 주소가 노션에 자동으로 다시 적힘

[스크린샷 자리 — 슬라이드 카드 목록·상태 뱃지]

Chapter 2.5

만들면서 배운 것 — 실제 시행착오

슬라이드 흐름·시리즈 구조·자료 정제·미리보기 — 5가지 시행착오를 거쳐 지금 구조가 나왔어요.

1
워크샵용 패턴만 만들었더니 웨비나·라이브가 안 됨
처음에는워크샵용 슬라이드 패턴(목차·챕터·본문·요약·마무리)만 만듦. 웨비나·라이브·세미나는 결이 전혀 다름 — 청중을 끌어들이고 설득하는 흐름이 따로 필요
→
바꿨더니슬라이드 흐름을 14장으로 다시 짬 — 인사 → 후킹 → 고민 → 공감 → 질문 → 컨셉 → 증거 → 숫자 → 사례 → 비포·애프터 → 시연 → 보상 예고 → 전환 → 다음 추천. 15분이든 60분이든 알아서 분배
2
"5주 시리즈" 식으로 묶었더니 청중이 부담
처음에는"SEO 마스터 5주" 식으로 1·2·3회차 묶음. 앞 회차 안 들으면 다음 이해가 어려워서, 중간 합류자는 진입 부담
→
바꿨더니한 번에 하나씩 끝나는 '세션' 단위로. 세션끼리는 "같이 들으면 좋은 세션" 연결만 둠. 청중은 관심 주제만 골라 듣고, 마지막 슬라이드에 다음 추천이 자동
3
노션 자료함에 정리된 글만 넣을 수 있다고 가정
처음에는노션 자료함에는 잘 정리된 글만 들어간다고 가정. 인터뷰·대화·메모 같은 날것은 슬라이드로 못 가져옴 — 매번 손으로 정리해서 다시 넣어야
→
바꿨더니날것을 자료함에 넣기 전 정제하는 단계를 따로 분리. 인터뷰·대화 → 핵심가치·페인·시행착오·비포애프터로 자동 구조화 → 자료함에 정돈된 페이지로
4
미리보기가 정사각형으로 나오고 줄바꿈도 깨짐
처음에는대시보드 미리보기를 16:9로 맞추는 표준 방법을 썼는데 환경에 따라 비율이 정사각형 가까이로 깨짐. 글자 줄바꿈도 풀스크린이랑 달라서 미리보기랑 실제가 안 맞음
→
바꿨더니미리보기 박스를 고정 크기로 두고 통째로 축소하는 방식. 어디서 봐도 정확히 16:9, 줄바꿈도 실제 발표 화면과 똑같음
5
노션에 회차·시리즈명 필수로 했더니 운영이 안 맞음
처음에는노션 일정함에 "회차"·"시리즈명" 필드를 필수로 둠. 모든 세션이 시리즈 구조에 강제로 들어가서, 매주 다른 주제로 운영하면 빈 값만 쌓임
→
바꿨더니"회차"·"시리즈명" 빼고 "유형"(세션·워크샵·세미나·특강·라이브) + "난이도"(기초·중급·심화) + "다음 추천 세션"으로 재설계. 매주 다른 주제로 가도 깔끔

결과 — 1세션 6시간 → 15분, 노션 한 흐름

매주 발표마다 매번 처음부터 만들던 흐름이, 노션 한 번 클릭으로 14장이 나오는 흐름으로 바뀌었습니다.

단순히 시간이 줄어든 것뿐 아니라, 노션·슬라이드·발표 후 정리가 한 흐름이라 "결과물이 어디 갔지?" 헤매는 일이 사라졌습니다. 청중에게도 다음 추천 세션이 자동으로 이어져요.

(2) 운영자 인터뷰

Q. 도입 전 가장 큰 어려움은?
보충 필요 예: "매주 발표인데 매번 슬라이드 14장을 손으로 만들었어요. 노션에 자료는 다 있는데 슬라이드로 옮기는 것만 4~5시간. 발표 끝나면 결과물도 흩어져서 다음 세션과 안 이어졌어요."
Q. 도입 후 가장 크게 달라진 점은?
보충 필요 예: "노션에서 세션 한 번 고르면 14장이 15분 안에 나와요. 모바일에서 미리 세션 정해두고 컴퓨터 앞에서 [생성] 한 번이면 끝. 발표 끝나면 슬라이드 주소까지 노션에 자동으로 적혀요."
Q. 14장 흐름은 어떻게 만들어졌나요?
보충 필요 예: "처음엔 워크샵 패턴만 만들었는데 웨비나·라이브가 안 됐어요. 5번 시행착오 거쳐서 인사→후킹→공감→증거→사례→시연→전환→다음 추천 14장으로 자리잡았어요."
Q. 어떤 사업자에게 추천하고 싶나요?
보충 필요 예: "매주 발표·웨비나·라이브 하시는 1인 운영자분들. 슬라이드 매번 손으로 만드는 시간만 줄여도 본업에 시간 크게 늘어나요."
